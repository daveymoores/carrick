The problem

Microservices are a black box—
even for your AI agent.

Your agent is brilliant inside a single repo. But the moment it needs to understand how services talk to each other, it's blind. You're back to manually tracing calls, grepping across repos, and hoping nothing breaks.

?

"Which service calls this endpoint?"

Your agent can see the route handler. It can't see the 4 other repos that depend on it. Change the contract and you'll find out in production.

⇄

"What types does this expect?"

The producer sends User. The consumer expects UserProfile. The mismatch sits quietly in your codebase until something blows up.

🔍

"Where's the business logic for X?"

It's split across three services, two shared libraries, and a Lambda. Good luck getting any AI to reason about that without a map.

How it works

One tool. Total cross-repo awareness.

Carrick scans every repo in your org, builds a live index of producers and consumers, and serves it to your agent via MCP. No schema files to maintain. No manual wiring.

Scan & extract

Carrick uses SWC to parse your Express services at the AST level. Every route, every mounted router, every call site—mapped automatically.

Resolve types

TypeScript types are resolved recursively across files using ts-morph. Request and response shapes are captured per-endpoint with full fidelity.

Index cross-repo

Endpoint metadata flows into a shared store. Every service sees every other service. Producers and consumers are linked across your entire org.

Serve via MCP

Your AI agent queries the Carrick MCP server and instantly knows what calls what, what types flow where, and what will break if you change something.

// Your agent asks Carrick via MCP: // "What depends on GET /api/users/:id?" Found 3 consumers across 3 repos: order-service → GET /api/users/:id ✓ types match billing-service → GET /api/users/:id ✗ missing field: role dashboard-app → GET /api/users/:id ✓ types match // Your agent now knows billing-service will break // before you even open a PR.

Features

Everything your agent needs to
reason across services.

MCP-native

Purpose-built as a Model Context Protocol server. Works with Claude, Cursor, Windsurf, and any MCP-compatible agent out of the box.

AST-level precision

No regex hacks. Carrick parses your code with SWC and ts-morph to extract routes, call sites, and types with compiler-grade accuracy.

Full type resolution

TypeScript interfaces resolved recursively. When your agent sees an endpoint, it sees the exact shape of every request and response.

GitHub Action built-in

Runs in CI on every PR. Catches cross-repo contract violations and posts actionable comments before code hits main.

AI-powered extraction

Complex call patterns—template literals, env vars, wrapper functions—are handled by an LLM agent that reads your code like a human would.

Incremental & fast

Only re-analyzes changed files. Results are cached. Even large monorepo-scale architectures get sub-minute analysis times.
